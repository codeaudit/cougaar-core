JVM stack dump support

This package includes:

  1. JniStackDump:
    The JNI implementation to signal a stack dump.

    The library name is "jvmdump". On Unix compiled file that must be in your $LD_LIBRARY_PATH (or -Djava.library.path) is "libjvmdump.so". On Windows the compiled file that must be in your %PATH% (or -Djava.library.path) is "jvmdump.dll".

    If you're using the AppServer, and want to use the "-Djava.library.path=", your "server.props" should include a "java.java.library.path=" line.

    This class is package private, but includes a "main(..)" method for testing.

  2. JvmStackDumpServiceComponent:
    A component which advertises the root-level "JvmStackDumpService" for use within the node. This is a service-based wrapper for the above JNI code.

  3. JvmStackDumpServlet:
    An optional component which loads a "/jvmdump" servlet, which uses the JvmStackDumpService.
